# Security Policy

Secracy takes security seriously.  
We welcome responsible disclosure of security vulnerabilities and are committed
to working with the security community to verify and address issues in a timely manner.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### üì© How to Report
- Email: **security@secracy.in**
- Subject line: `Security Vulnerability Report`

Please include:
- A clear description of the vulnerability
- Steps to reproduce (proof-of-concept if possible)
- Impact assessment (what an attacker could achieve)
- Any relevant screenshots, logs, or references

### üîê What NOT to Do
- Do not publicly disclose the vulnerability before it is fixed
- Do not exploit the vulnerability beyond what is necessary to demonstrate risk
- Do not perform denial-of-service (DoS) attacks or social engineering

## Response Timeline

We aim to:
- Acknowledge receipt within **48 hours**
- Provide a status update within **5‚Äì7 business days**
- Resolve valid vulnerabilities as quickly as possible based on severity

Timelines may vary depending on complexity and impact.

---

## Disclosure Policy

- If a vulnerability is accepted, we will work on a fix and deploy it promptly
- If declined, we will explain the reasoning clearly
- Coordinated disclosure is encouraged and appreciated

At this time, Secracy does not offer a public bug bounty program, but
responsible researchers will be acknowledged when appropriate.

---

## Scope

This policy applies to:
- https://secracy.in
- All publicly accessible assets owned and operated by Secracy

Third-party services and dependencies should be reported to their respective owners.

---

## Safe Harbor

Secracy supports good-faith security research conducted under this policy.
We will not pursue legal action against researchers who:
- Follow this policy
- Avoid data destruction or privacy violations
- Report findings responsibly
